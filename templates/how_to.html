На главную

  1. Первым делом нужно зарегистрироваться. Все пароли я храню в зашифрованном виде и имею доступ только к хешам. Смена пароля пока не предусмотрена, но если вам очень надо - можете попросить меня удалить вашего пользователя из базы и создать нового с новым паролем.
  2. Для того чтобы войти вам нужно дождаться подтверждения администратора. Пока подверждать может только
    Владислав Невструев.
  3. До подтверждения вы имеете доступ только к этой странице, списку тегов, страницам авторизации и регистрации. После подтверждения вам доступен весь имеющийся функционал.
Работа с тегами
  1. Чтобы добавить тег, вы можете на главной странице нажать кнопку Добавить тег
  2. Вы можете редактировать теги. Для этого в Списке тегов необходимо нажать кнопку Редактировать. Откроется такая же форма, как и при добавлении тега, но с уже заполненными полями, которые вы можете как угодно исправлять.
  3. При добавлении тега есть следующие поля:
    1. Тег. Название тега на
      английском языке.
    2. Родитель. У каждого тега, кроме самых основных, есть
      один
      родитель. Например, дерево отрезков - это структура данных, поэтому тег "Дерево отрезков" - ребенок тега "Структуры данных". Если вы добавляете новый основной тег, то не указывайте у него родителя. Иначе выберите наиболее подходящего родителя из списка.
Работа с задачами
  1. Чтобы добавить задачу, вы можете на главной странице нажать кнопку Добавить задачу. Я не рекомендую такой способ добавления задачи, поскольку их потом будет несколько труднее собирать в контест. Рекомендуемый способ смотрите в работе с контестами.4.2. Однако, если вы пока не хотите, чтобы новая задача содержалась в каком-либо контесте, то это единственный способ добавить задачу.
  2. Также задачу можно редактировать. Для этого на странице задачи, например здесь требуется нажать на кнопку Редактировать задачу. Откроется такая же форма, как и при добавлении задачи, но с уже заполненными полями, которые вы можете как угодно исправлять.
  3. При добавлении задачи есть следующие поля:
    1. Название. Тут все понятно.
      Поле обязательное
    2. Короткое условие. Здесь следует коротко написать, что надо сделать в задаче. Желательно обойтись 1-2 предложениями.
      Поле обязательное
    3. Полное условие. Пока я сюда просто копирую условие из Яндекс.Контеста. В будущем добавлю возможность загрузки файла. В любом случае, смысл этого поля - показать условие так, как его видят участники контеста.
      Поле не обязательное
    4. Разбор. Тут можно написать решение. В будущем добавиться возможность загружать файл.
      Поле не обязательное
    5. Примерная сложность. Здесь я пишу число от 1 до 100, насколько сложной мне кажется задача. Это очень субъективное понятие, которое у меня в разные дни бывает разным. В будущем мы придумаем какую-нибудь метрику, пока ставьте по
      своему усмотрению
      .
      Поле не обязательное, по умолчанию 0
    6. Источник. Здесь мы указываем, откуда Снарк может взять эту задачу для наших нужд. Если задача с Яндекс.Контеста, то достаточно написать ЯК <номер контеста> <номер задачи>. Важно, чтобы по источнику можно было однозначно добраться до задачи.
      Поле обязательное
    7. Todo. Какие-либо косяки, замеченные в задаче. В основном, слабые тесты, пропускающие долгие решения. Бывают опечатки в условии, ненаписанный разбор и прочее.
      Поле не обязательное
    8. Теги. То, ради чего делалась база. Вы можете добавить тег, нажав на кнопку Добавить тег. После этого появится еще одно поле для тега. Тег нужно выбрать в
      выпадающем списке.
      Если вы добавили слишком много полей, можно их убрать кнопкой Удалить. Все теги представлены в виде дерева. Если вы выбрали какой-либо тег, нет смысла выбирать всех его родителей - они появятся в базе автоматически.
      Поле обязательное, необходим хотя бы один непустой тег
Работа с контестами
  1. Чтобы добавить контест, вы можете на главной странице нажать на кнопку Добавить контест.
  2. Вы можете редактировать контест. Для этого на странице контеста, например здесь требуется нажать на кнопку Редактировать. Откроется такая же форма, как и при добавлении контеста, но с уже заполненными полями, которые вы можете как угодно исправлять.
  3. Вы также можете копировать контесты. Для этого требуется нажать на кнопку Копировать. Откроется такая же форма, как и при редактировании контеста, но при отправке создастся новый, а не отредактируется старый. Этот функционал нужен для быстрого добавления контестов из Летних и Зимних олимпиадных школ, которые довольно похожи друг на друга.
  4. В контест можно добавлять задачи. Есть два пути, как это можно сделать:
    1. Добавить уже существующую в базе задачу. Это можно сделать через редактирование контеста и добавления задачи внизу страницы
    2. Добавить новую задачу в базу. Для этого на странице контеста требуется нажать на кнопку Добавить новую задачу. Откроется интерфейс добавления новой задачи, однако при добавлении задача автоматически окажется в новом контесте. Это
      рекомендуемый
      способ добавлять задачи в базу.
  5. При добавлении контеста есть следующие поля:
    1. Название. Тут все понятно.
      Поле обязательное
    2. Год. Год проведения контеста.
      Поле обязательное
    3. Описание. Тут требуется пояснить, что это за контест. Является ли он тематическим, либо каким-либо официальным соревнованием.
      Поле обязательное
    4. Ссылка на контест. В основном, я указываю здесь ссылку на таблицу результатов контеста, но подойдет также любая ссылка, через которую можно добраться до контеста.
      Поле обязательное
    5. Ссылка на разбор. Я планирую в будущем поменять ее на загрузку файла разбора.
      Поле не обязательное, но вставить можно только корректную ссылку
    6. Список задач. Здесь можно выбрать, какие задачи из имеющихся в базе вы хотели бы добавить в этот контест. Для добавления задач, которых в базе пока нет, воспользуйтесь рекомендуемым методом. В частности, вы можете оставить этот список пустым, если все задачи из контеста новые, однако для этого требуется явно удалить изначальное пустое поле из списка.
Просмотр содержимого
  1. Для просмотра списка тегов на главной странице нажмите кнопку Список тегов. Он отобразится в виде дерева тегов. Рядом с каждым тегом появится кнопка для его редактирования.
  2. Для просмотра списка задач на главной странице нажмите кнопку Список задач. Он отобразится в виде списка задач. Рядом с каждой задачей появится кнопка для просмотра страницы задачи. Это единственный способ увидеть полное условие задачи. Также есть возможность фильтровать задачи по тегам. Для этого сверху страницы есть фильтр, который демонстрирует только те задачи, которые имеют все запрошенные вами теги. Для сброса фильтра сделайте запрос с пустым набором тегов.
  3. Для просмотра списка контестов на главной странице нажмите кнопку Список контестов. Он отобразится в виде списка контестов. Рядом с каждым контестом появится ссылка на этот контест, разбор контеста, а также ссылку на страницу контеста и копированя контеста. На странице контеста отобразится список задач, которые в него входят, а также ссылки на контест, разбор, добавления новой задачи, редактирования и копирования контеста.
